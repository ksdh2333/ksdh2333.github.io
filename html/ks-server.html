Ks-server

介绍

一个 开在自己电脑上 时不时就会掉线重启的服务端, 还有可能被Ks-game影响。

开服务端原本只为了使用MCDReforged的插件和Velocity互通服的方便, 但是也可以用来联机。
但是通常没有人会想要加入, 一是服务器 不稳定 , 二是不会为了玩游戏而耗费过多精力。

如果有人加入服务器, 那么这个服务器其实就是是朋友联机,
所有人为了 同一个目标 (比如建造机器) 努力。

服务器现有 3 个分服, main(主服), mirror(镜像服), crea(创造服)

主服
生存主服, 生电服本体
镜像服
主服 的镜像, 不定期同步 主服 存档, 模式为 创造 , 用于刷怪塔 测试 、坐标 测试 等。请不要将 镜像服 与 主服 搞混
创造服
虚空存档, 创造模式 , 用于机器、建筑 设计测试

*生存主服 死亡掉落

插件/模组

服务端现为 Java 版 1.21fabric , 使用 MCDReforged 代理, 并使用 Velocity 建立服务器联通

  • Velocity插件

    maintenance
    为服务器添加 维护模式 , 启用维护模式时, 普通玩家不能进入服务器
    minimotd
    为服务器在服务器列表的 MOTD文字 添加彩色等效果
    Reconnect
    将玩家 重新连接 到上次退出前连接的子服

  • MCDReforged插件

    Beep
    为服务器添加 @玩家 的功能
    CrashRestart
    服务端 崩溃时自动重启
    DayCountNBT
    !!day可以查看服务器 已开服时间
    MirrorMcsmcdR(仅 主服 )
    用于 自动同步 主服 存档到 镜像服
    PrimeBackup
    全图备份 插件, 每天定时备份, !!pb make手动创建备份
    Region-BackUp
    局部备份 插件, 主服 很少使用, 会回档地形但不回档玩家
    simple_op
    !!op以获取2级服务器 OP 权限 (仅 镜像服 、 创造服 )

  • fabric模组

    carpet, Carpet-AMS-Addition, carpet-extra, carpet-org-addition, carpet-tis-addition, CrystalCarpetAddition, gugle-carpet-addition
    为创造设计提供便捷性功能, 重新引入更新抑制、旧版袭击机制, 添加假人等功能
    XaerosWorldMap, Xaeros_Minimap
    让客户端 XaerosWorldMap 、Xaeros_Minimap 更兼容, 并能无视距离显示其他玩家位置
    Jade
    让客户端 Jade 更兼容, 能获取到蜜蜂等服务端信息
    Lithium
    优化服务端 Mspt
    noisium
    优化 区块加载和生成 造成的卡顿
    pca-protocol
    让客户端 MasaGadget 更兼容, 能获取到村民相关信息等服务端信息
    quickrightclick, Advanced Shulkerboxes
    让玩家可以 对空气右键打开 潜影盒、工作台以及末影箱
    servux
    让客户端 Masa 系列模组更兼容, 能获取到结构范围、史莱姆区块等服务端信息
    syncmatica
    用于与客户端 Syncmatica 配合 共享投影原理图

规则

客户端功能

仅服主 允许的客户端功能 可以使用, 任何没有经过许可的客户端功能需要 向服主询问 。
不允许使用矿透 , 仅允许世吞维护助手。
与一些人认为 都玩生电了还管矿透? 不同, 我认为 都玩生电了还缺那点矿?
不允许投影打印机 。
恶意破坏服务器、卡服将直接加入黑名单, 无特殊情况永久不移除。

假人使用

玩家可以任意使用 <玩家自己的id> + <大于3的数字> 形式的假人;
<玩家自己的id> + "1" 为其他玩家向此玩家传输属于此玩家的物品时使用, 如死亡后的尸体;
<玩家自己的id> + "2" 为该玩家向其他玩家传输属于自己的物品时使用;
<玩家自己的id> + "3" 为该玩家需要临时清空背包时将自己的物品寄存的假人。
除 <玩家自己的id> + <数字> 形式以外的所有假人都要在假人注册表中按格式注册。
机器挂机假人通常为 <机器名> + <数字> 形式,
若机器名称过长, 则为 <主产物名> + "_" + <数字> 形式
存储用假人为 <存储的物品名> + <数字> 形式, 存储假人头盔位置应戴有存储的物品一个

其他规则由服主或管理员解释。
